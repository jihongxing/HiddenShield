# 图片音频水印鲁棒性基准

这个基准用于评估图片/音频写入后的可验证性，重点观察水印在常见二次处理后的提取稳定性。

音频写入基准默认使用 30 秒样本。短于 30 秒的音频不作为 HiddenShield 音频版权保护的产品承诺范围；短裁剪片段只作为取证扩展观察项。

## 运行方式

```powershell
npm run watermark:bench -- --image-dir "E:\Users\jihx\Pictures\人脸" --audio-glob "E:\Users\jihx\Pictures\*.mp3"
```

默认只取前 3 张图片和前 3 个音频，输出目录为：

```text
watermark-core/target/robustness-bench/run-<timestamp>/
```

其中：

- `report.md`：人工阅读报告
- `report.json`：后续自动化分析用报告
- `images/`：每个图片样本的写入结果和变换结果
- `audio/`：每个音频样本的 WAV 中间文件和变换结果

## 覆盖场景

图片：

- 基线 PNG
- PNG 重新编码
- JPEG 90 质量
- JPEG 75 质量
- 缩放到 85%
- 裁剪 2%

音频：

- 基线 WAV
- WAV 重新编码
- 音量 80%
- 音量 120%
- 重采样到 22050 Hz
- MP3 192k 转码后再转回 WAV

## 参数

```powershell
npm run watermark:bench -- `
  --image-dir "E:\Users\jihx\Pictures\人脸" `
  --audio-glob "E:\Users\jihx\Pictures\*.mp3" `
  --max-images 10 `
  --max-audio 10
```

可选参数：

- `--max-images <N>`：限制图片样本数
- `--max-audio <N>`：限制音频样本数
- `--output-dir <dir>`：自定义报告输出目录
- `--ffmpeg <path>`：指定 FFmpeg 路径

## 判定规则

每个变换都会重新提取水印，并与写入时的 `watermark_uid` 对比：

- `PASS`：能提取，且 UID 一致
- `FAIL`：提取失败，或提取出的 UID 与期望不一致

这个工具只读取原始素材，不会覆盖素材目录里的文件。
