# 技术债：视频画面水印 (Video Frame Watermark)

## 状态：挂起 (Post-MVP)

## 背景

当前视频水印仅嵌入音轨（QIM 频域盲水印）。如果侵权方去除音轨只取画面，水印将丢失。

## 未来方案

在视频帧的 DCT 中频系数或运动向量中嵌入帧级水印，实现双通道冗余（音轨 + 画面）。

### 技术挑战

- 逐帧 DCT/DWT 计算量极大，本地 CPU 处理速度不可接受
- GPU Shader 实现工程量翻倍
- 需要在编码前注入，与 FFmpeg pipeline 深度集成

### 建议路线

作为 Pro 版（云端重火力加密）的核心卖点：
- 免费版：音轨水印（本地处理）
- Pro 版：音轨 + 画面双通道水印（云端 GPU 集群处理）

## 优先级

低 — MVP 阶段音轨水印已覆盖 90%+ 的实际侵权场景（大多数盗搬不会去音轨）。
