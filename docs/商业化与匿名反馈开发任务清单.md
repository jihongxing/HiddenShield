# HiddenShield 商业化与匿名反馈开发任务清单

## Phase 0：对齐基础口径（已完成）

1. 定义收费与匿名统计的产品边界。
   - 验收：确认订阅主导、点数预留、匿名统计明确告知且可关闭。
2. 更新产品文档、隐私/协议文案与前台提示。
   - 验收：对外口径统一为免费期优先、订阅主线、点数预留、匿名反馈低打扰。

## Phase 1：本地权益与用量账本（已完成）

1. 新增权益状态模型。
   - 包含 `free`、`trial`、`active`、`grace`、`expired`。
   - 验收：任意功能可依据权益状态判断是否允许执行。
2. 新增用量账本表。
   - 记录一次成功处理的图片/视频/音频事件。
   - 验收：失败、取消、异常退出不入账。
3. 新增订阅占位字段。
   - 包括订阅来源、有效期、到期时间、最后一次校验时间。
   - 验收：先不接支付也能完整跑状态机。

## Phase 2：匿名反馈管道（已完成）

1. 设计匿名 install/session id。
   - 验收：首次安装后生成并持久化，重装后可重新生成。
2. 建立本地匿名事件队列。
   - 验收：离线时可缓存，联网后批量上报。
3. 定义标准事件模型。
   - 包含功能名、成功/失败、错误码、耗时、桶化媒体信息、模块版本、堆栈摘要。
   - 验收：payload 不含文件名、路径、hash、原始 bytes。
4. 增加失败诊断包生成器。
   - 验收：崩溃或失败事件可自动附带脱敏诊断信息。
5. 增加 flush 逻辑。
   - 验收：退出时、切后台后、网络恢复时均可尝试发送。

## Phase 3：后端上报与存储（已完成）

1. 定义匿名统计上传接口契约。
   - 验收：客户端和服务端共享同一事件 schema。
2. 实现批量上报。
   - 验收：网络抖动不影响本地处理主流程。
3. 实现重试和退避。
   - 验收：连续失败后自动降频，避免空转。
4. 实现服务端接收与落库。
   - 验收：可按版本、功能、错误码、日期统计。

## Phase 4：UI 与交互（已完成）

1. 首启告知与确认文案更新。
   - 验收：用户能看懂匿名统计用途与控制方式。
2. 设置页新增统计概览。
   - 验收：能查看本机发送次数、失败次数、待发送队列数量。
3. 设置页新增“发送诊断”入口。
   - 验收：失败后用户可一键手动提交诊断包。
4. 设置页新增订阅入口。
   - 验收：能看到当前权益状态、试用状态、到期时间和升级路径。

## Phase 5：埋点与风控（已完成）

1. 给核心功能埋点。
   - 工作台启动、图片处理、音频处理、视频处理、验证、导出、失败等。
   - 验收：每条主路径都有成功/失败事件。
2. 给权益判断埋点。
   - 验收：能识别 free、trial、active、grace、expired 的分布。
3. 给异常增加去重和速率限制。
   - 验收：同类错误不会刷屏上报。
4. 增加基础风控指标。
   - 验收：能看出异常重试、崩溃率、失败率、转化漏斗。

## Phase 6：验证与测试（已完成）

1. 队列离线测试。
   - 验收：断网状态下可积压，联网后恢复发送。
2. 退出 flush 测试。
   - 验收：关闭应用前待发送事件尽可能发送成功。
3. 脱敏测试。
   - 验收：事件中不出现文件路径、原始 hash、媒体内容。
4. 账本一致性测试。
   - 验收：成功 1 次记 1 次，失败不记账。
5. 权益状态机测试。
   - 验收：试用、到期、宽限期、恢复订阅都能正确切换。
6. 崩溃回传测试。
   - 验收：panic/FFmpeg 异常可生成可读诊断包。

## Phase 7：上线顺序（待执行）

1. 先上线匿名反馈与账本。
2. 再上线订阅状态机与权益判断。
3. 最后开放真实支付与点数包。
