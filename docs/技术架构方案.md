# 隐盾 MVP 技术架构方案：Tauri + Rust 桌面端

**架构定位：** 轻量级桌面客户端，Rust 后端驱动 FFmpeg 原生进程，前端仅负责 UI 展示。  
**核心优势：** 无浏览器沙箱限制、原生文件系统访问、GPU 硬件加速、二进制逆向难度高。

---

## 一、技术栈总览

| 层级 | 技术选型 | 职责 |
|------|---------|------|
| 前端 UI | Vue 3 + Vite + TypeScript | 文件拖拽、进度展示、版权库浏览 |
| 桌面框架 | Tauri 2.x | 窗口管理、系统 API、自动更新、IPC 通信 |
| 核心后端 | Rust | FFmpeg 进程调度、水印算法、SQLite 操作、硬件探测 |
| 音视频处理 | FFmpeg 7.x（原生二进制） | 视频解封装/编码/压制/色彩转换 |
| 水印算法 | Rust（rustfft / realfft） | 音频频域盲水印嵌入与提取 |
| 本地存储 | SQLite（通过 rusqlite） | 版权金库、用户配置、处理历史 |
| 元数据探测 | ffprobe（FFmpeg 附带） | 源视频分辨率/帧率/色彩空间/时长 |

## 四、FFmpeg 集成策略

### 4.1 FFmpeg 二进制管理

**方案：首次启动自动下载 + 本地缓存**

```rust
/// 检查 FFmpeg 是否可用，不可用则自动下载
pub async fn ensure_ffmpeg() -> Result<PathBuf, FfmpegError> {
    let ffmpeg_dir = app_data_dir().join("ffmpeg");
    let ffmpeg_bin = if cfg!(target_os = "windows") {
        ffmpeg_dir.join("ffmpeg.exe")
    } else {
        ffmpeg_dir.join("ffmpeg")
    };
    
    if ffmpeg_bin.exists() {
        return Ok(ffmpeg_bin);
    }
    
    // 从 CDN 下载预编译的 FFmpeg（GPL 静态链接版本）
    // Windows: ~80MB, macOS: ~60MB
    download_ffmpeg(&ffmpeg_dir).await?;
    
    Ok(ffmpeg_bin)
}
```

**为什么不内嵌到安装包：**
- FFmpeg 二进制 60-80MB，会让安装包从 ~15MB 膨胀到 ~90MB
- 分离后可以独立更新 FFmpeg 版本
- 用户如果已安装系统 FFmpeg，可以直接复用

### 4.2 FFmpeg 进程管理

```rust
use tokio::process::Command;
use tokio::io::{AsyncBufReadExt, BufReader};

pub struct FfmpegProcess {
    child: tokio::process::Child,
    total_duration_secs: f64,
}

impl FfmpegProcess {
    pub async fn spawn(args: &[&str], duration: f64) -> Result<Self, Error> {
        let child = Command::new(get_ffmpeg_path())
            .args(args)
            .stderr(std::process::Stdio::piped())  // FFmpeg 进度输出在 stderr
            .stdout(std::process::Stdio::null())
            .spawn()?;
        
        Ok(Self { child, total_duration_secs: duration })
    }
    
    /// 实时解析 FFmpeg 进度（从 stderr 读取 time= 字段）
    pub async fn stream_progress(&mut self) -> impl Stream<Item = f64> {
        let stderr = self.child.stderr.take().unwrap();
        let reader = BufReader::new(stderr);
        let duration = self.total_duration_secs;
        
        // FFmpeg stderr 输出格式：
        // frame= 1234 fps= 60 ... time=00:01:23.45 bitrate=8000kbps ...
        // 解析 time= 字段计算百分比
        async_stream::stream! {
            let mut lines = reader.lines();
            while let Ok(Some(line)) = lines.next_line().await {
                if let Some(time_str) = extract_time(&line) {
                    let current_secs = parse_time_to_secs(&time_str);
                    let progress = (current_secs / duration).min(1.0);
                    yield progress;
                }
            }
        }
    }
}
```

## 五、盲水印算法模块

### 5.1 技术原理

采用音频频域扩频水印（Spread Spectrum Watermarking）：
1. 将音频分帧（每帧 4096 采样点）
2. 对每帧执行 FFT 变换到频域
3. 在中高频段（2kHz - 8kHz）微调幅度，编码水印比特
4. 执行 IFFT 还原时域信号
5. 听觉掩蔽：静音段跳过，避免底噪

### 5.3 水印嵌入核心逻辑（简化）

```rust
use realfft::RealFftPlanner;

const FRAME_SIZE: usize = 4096;
const WATERMARK_BAND_LOW: usize = 180;   // ~2kHz @ 44100Hz
const WATERMARK_BAND_HIGH: usize = 740;  // ~8kHz @ 44100Hz
const EMBED_STRENGTH: f32 = 0.02;        // 嵌入强度（不可闻阈值）
const SILENCE_THRESHOLD: f32 = 0.001;    // 静音段跳过阈值

pub fn embed_watermark(audio_samples: &mut [f32], watermark_bits: &[u8]) -> Result<(), WatermarkError> {
    let mut planner = RealFftPlanner::<f32>::new();
    let fft = planner.plan_fft_forward(FRAME_SIZE);
    let ifft = planner.plan_fft_inverse(FRAME_SIZE);
    
    let mut bit_index = 0;
    
    for frame in audio_samples.chunks_mut(FRAME_SIZE) {
        if frame.len() < FRAME_SIZE {
            break;
        }
        
        // 静音检测：跳过能量过低的帧
        let energy: f32 = frame.iter().map(|s| s * s).sum::<f32>() / FRAME_SIZE as f32;
        if energy < SILENCE_THRESHOLD {
            continue;
        }
        
        // FFT
        let mut spectrum = fft.make_output_vec();
        let mut input = frame.to_vec();
        fft.process(&mut input, &mut spectrum).unwrap();
        
        // 在水印频段嵌入比特
        let bit = watermark_bits[bit_index % watermark_bits.len()];
        let sign = if bit == 1 { 1.0 } else { -1.0 };
        
        for k in WATERMARK_BAND_LOW..WATERMARK_BAND_HIGH {
            let magnitude = spectrum[k].norm();
            spectrum[k] += spectrum[k] / magnitude * EMBED_STRENGTH * sign;
        }
        
        // IFFT
        let mut output = ifft.make_output_vec();
        ifft.process(&mut spectrum, &mut output).unwrap();
        
        // 归一化并写回
        let norm_factor = 1.0 / FRAME_SIZE as f32;
        for (i, sample) in frame.iter_mut().enumerate() {
            *sample = output[i] * norm_factor;
        }
        
        bit_index += 1;
    }
    
    Ok(())
}
```

### 5.4 水印数据格式

嵌入的水印内容为固定 256 bit（32 字节）：

```
[4 bytes: magic number 0xHD5H]  // 用于验证是否为隐盾水印
[8 bytes: 时间戳 (Unix epoch)]   // 精确到秒
[16 bytes: 机器指纹 SHA-256 前16字节]
[4 bytes: CRC32 校验]
```

## 七、安全与逆向防护

| 威胁 | 应对措施 |
|------|---------|
| 水印算法被逆向 | Rust 编译为原生二进制，逆向难度远高于 JS/Wasm。关键参数（频段、强度）可混淆。 |
| 水印被针对性破坏 | 扩频水印天然抗噪声攻击。多频段冗余编码提高鲁棒性。 |
| 用户数据泄露 | 全本地处理，无网络传输。SQLite 数据库可选加密（sqlcipher）。 |
| FFmpeg 供应链攻击 | 下载时校验 SHA-256，使用官方 release 或自编译版本。 |

## 八、与旧方案的对比

| 维度 | 旧方案（Web + Wasm） | 新方案（Tauri + Rust） |
|------|---------------------|----------------------|
| 文件大小限制 | 500MB（浏览器内存） | 无限制（流式处理） |
| 处理速度 | 慢（Wasm 单线程） | 极快（原生多线程 + GPU） |
| 内存占用 | 文件大小 × 3-4 倍 | 可控（流式读写） |
| 逆向难度 | 低（JS/Wasm 可读） | 高（原生二进制） |
| 跨域配置 | 需要 COOP/COEP | 不需要 |
| 离线可用 | 需要 Service Worker | 天然离线 |
| 安装门槛 | 无（网页） | 需下载安装（~15MB） |
| GPU 加速 | 不可能 | 完全支持 |
| 4K 视频处理 | 不可能 | 完全支持 |
