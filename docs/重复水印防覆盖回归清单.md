# 重复水印防覆盖回归清单

用于验证图片、音频默认防覆盖，以及显式重写后的金库链路是否正确。

---

## 1. 前置准备

- 准备一张 PNG 或 JPG 测试图片。
- 准备一个 WAV 测试音频。
- 确认创作者身份已初始化。
- 清空或记录当前版权库条目数，方便对比新增记录。

---

## 2. 图片默认防覆盖

步骤：

1. 在工作台选择测试图片。
2. 不勾选“允许重写已有隐盾水印”。
3. 开始处理，生成第一份带水印图片。
4. 再次选择刚生成的带水印图片。
5. 仍然不勾选“允许重写已有隐盾水印”。
6. 再次开始处理。

预期：

- 第一次处理成功。
- 第二次处理失败。
- 错误信息能表达“已有隐盾水印”或同等含义。
- 版权库只新增第一次成功记录。
- 第一条记录 `revision = 1`。
- 第一条记录 `parent_watermark_uid = null`。
- 第一条记录 `rewrite_reason = null`。

---

## 3. 图片显式重写

步骤：

1. 选择第 2 节生成的带水印图片。
2. 勾选“允许重写已有隐盾水印”。
3. 填写重写原因，例如 `图片修正版重新导出`。
4. 确认风险弹窗。
5. 开始处理。
6. 打开版权库查看新增记录。
7. 使用取证页检测新输出图片。

预期：

- 处理成功。
- 版权库新增一条记录。
- 新记录 `revision = 2`。
- 新记录 `parent_watermark_uid` 等于第一次图片记录的 UID。
- 新记录 `rewrite_reason = 图片修正版重新导出`。
- 版权库“重写链路”区域展示父子 UID。
- 取证页命中新记录，并展示写入版本和父级 UID。

---

## 4. WAV 默认防覆盖

步骤：

1. 在工作台选择测试 WAV。
2. 不勾选“允许重写已有隐盾水印”。
3. 开始处理，生成第一份带水印 WAV。
4. 再次选择刚生成的带水印 WAV。
5. 仍然不勾选“允许重写已有隐盾水印”。
6. 再次开始处理。

预期：

- 第一次处理成功。
- 第二次处理失败。
- 版权库只新增第一次成功记录。
- 第一条音频记录 `revision = 1`。
- 第一条音频记录 `parent_watermark_uid = null`。
- 第一条音频记录 `rewrite_reason = null`。

---

## 5. WAV 显式重写

步骤：

1. 选择第 4 节生成的带水印 WAV。
2. 勾选“允许重写已有隐盾水印”。
3. 填写重写原因，例如 `音频授权派生版`。
4. 确认风险弹窗。
5. 开始处理。
6. 打开版权库查看新增记录。
7. 使用取证页检测新输出 WAV。

预期：

- 处理成功。
- 版权库新增一条记录。
- 新记录 `revision = 2`。
- 新记录 `parent_watermark_uid` 等于第一次音频记录的 UID。
- 新记录 `rewrite_reason = 音频授权派生版`。
- 取证页命中新记录，并展示写入版本和父级 UID。

---

## 6. 视频入口保护

步骤：

1. 选择一个视频文件。
2. 查看工作台高级选项。

预期：

- 视频流程不展示“允许重写已有隐盾水印”。
- 视频压制仍按原流程工作。
- 视频记录默认 `revision = 1`。
- 视频记录没有父级 UID。

---

## 7. 摘要与报告

步骤：

1. 对显式重写后的图片或音频执行取证。
2. 点击复制报告或分享摘要。

预期：

- 摘要包含水印 UID。
- 摘要包含写入版本。
- 有父级记录时摘要包含父级 UID。
- 有重写原因时摘要包含重写原因。

---

## 8. 自动化验证

提交前至少运行：

```bash
cargo test
```

工作目录：

- `watermark-core`
- `src-tauri`

前端至少运行：

```bash
npm run build
```

---

## 9. 通过标准

- 默认重复写入必须失败。
- 显式重写必须成功。
- 金库必须记录父级 UID、写入版本、重写原因。
- 取证页必须能展示链路信息。
- 视频入口不能开放重写选项。
