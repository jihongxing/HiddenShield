# HiddenShield 统一账户与云同步重设计方案

## 结论

移动端与桌面端的正式同步方案应改为“统一账户 + 云同步服务”。当前已经实现的“移动端直连桌面端 HTTP 同步”不应作为主产品路径继续打磨，只保留为开发联调、局域网调试或高级导入桥接能力。

正确产品口径：

- 不登录也能本地使用。
- 要跨设备同步，必须登录 HiddenShield 账户并开启云同步。
- 移动端和桌面端都与平台云服务同步，不互相充当对方的服务端。
- 原始媒体默认不上传，首期只同步版权库元数据、取证记录、水印链路和审计记录。

## 为什么要重设计

当前桌面直连方案的问题不是实现细节，而是产品架构方向不对：

- 用户必须同时打开手机和电脑，使用条件脆弱。
- 局域网、防火墙、端口、配对码都会成为同步失败点。
- 没有统一用户身份，无法稳定回答“这条版权记录属于谁”。
- 换机、丢设备、多设备、多桌面、多手机都难以处理。
- 审计、恢复、撤销、团队权限、订阅权益无法自然挂接。
- 移动端和桌面端互相依赖，破坏“本地可独立使用”的产品体验。

因此，直连桌面端只能是调试能力，不是成熟产品同步能力。

## 新产品模型

### 本地模式

用户不登录也可以使用：

```text
移动端 App -> 本地 SQLite
桌面端 App -> 本地 SQLite
```

本地模式支持：

- 图片盲水印写入 / 提取。
- WAV 音频盲水印写入 / 提取。
- 本地版权库。
- 本地取证记录。
- 本地搜索、筛选、详情。

本地模式不支持：

- 跨设备自动同步。
- 换机恢复。
- 云端审计备份。
- 团队共享。

### 云同步模式

用户登录并开启同步后：

```text
移动端 App ─┐
           ├─ HiddenShield Cloud Sync API ─ 账户 / 设备 / 版权库 / 取证 / 审计
桌面端 App ─┘
```

云同步模式支持：

- 移动端和桌面端版权库漫游。
- 多设备同步。
- 失败重试。
- 自动解决审计。
- 换机恢复。
- 后续订阅权益、团队空间、云端视频处理。

### 局域网调试模式

当前 `DesktopHttpSyncTransport` 保留，但改名和降级：

```text
移动端 App -> 桌面端本地 HTTP 服务
```

用途：

- 开发联调。
- 内部测试。
- 用户高级导入 / 临时迁移。
- 无云环境下的工程验证。

不作为默认入口，不出现在主同步路径里。

## 身份体系

### Account Identity

平台账户身份，是云同步的主归属。

字段建议：

- `account_id`
- `email` / `phone` / 第三方登录标识
- `display_name`
- `created_at`
- `status`

### Workspace

首期可以每个账户默认一个个人空间，后续支持团队。

字段建议：

- `workspace_id`
- `account_id`
- `name`
- `role`
- `created_at`

### Device Identity

每台设备独立生成，不共享。

字段建议：

- `device_id`
- `account_id`
- `device_name`
- `platform`
- `app_version`
- `public_key`
- `last_seen_at`
- `created_at`

### Creator Identity

水印创作者身份，负责水印归属语义。

建议从“本机创作者标签”升级为账户下的创作者档案：

- `creator_id`
- `account_id`
- `display_name`
- `creator_seed_ref`
- `created_at`

首期不需要把原始 seed 明文上传。可以本地保存 seed，云端保存加密包或 seed 派生标识。

## 客户端同步模式

移动端和桌面端都统一为：

```text
SyncMode.localOnly
SyncMode.cloud
SyncMode.lanDebug
```

### localOnly

- 默认模式。
- 不要求登录。
- 所有记录只保存在本机。
- 同步队列可以存在，但不会发送。

### cloud

- 登录账户后可启用。
- 同步队列发送到云端。
- 拉取云端变更。
- 支持设备游标。

### lanDebug

- 当前桌面 HTTP 同步能力。
- 仅开发 / 高级设置可见。
- 不承诺稳定用户体验。

## 同步数据范围

首期云同步只同步元数据，不同步原始媒体：

- 版权记录。
- 水印 UID。
- revision / 写入次数。
- 原始文件 hash。
- 父水印 UID。
- 重写原因。
- 取证记录。
- 提取时间戳。
- 提取设备 ID。
- 提取文件 hash。
- 自动解决审计。
- 客户端队列状态摘要。

默认不同步：

- 原始图片。
- 原始音频。
- 原始视频。
- 本地文件路径。

后续可做可选加密媒体备份，但必须单独授权。

## API 设计

### Auth

```http
POST /v1/auth/login
POST /v1/auth/refresh
POST /v1/auth/logout
GET  /v1/me
```

首期可选登录方式：

- 邮箱验证码。
- 手机验证码。
- Apple / Google 后续补。

### Device

```http
POST /v1/devices/register
GET  /v1/devices
PATCH /v1/devices/{deviceId}
DELETE /v1/devices/{deviceId}
```

### Push

```http
POST /v1/sync/events:batch
Authorization: Bearer <token>
```

请求：

```json
{
  "deviceId": "device_mobile_001",
  "events": [
    {
      "clientEventId": "1700000000000-upsertVaultRecord",
      "operation": "upsertVaultRecord",
      "entityType": "vaultRecord",
      "entityId": "local-record-1",
      "payload": {}
    }
  ]
}
```

响应：

```json
{
  "accepted": 1,
  "acceptedEventIds": ["1700000000000-upsertVaultRecord"],
  "resolutions": []
}
```

### Pull

```http
GET /v1/sync/changes?cursor=...
Authorization: Bearer <token>
```

响应：

```json
{
  "nextCursor": "cursor_2026_06_16_0001",
  "changes": [
    {
      "entityType": "vaultRecord",
      "operation": "upsert",
      "entity": {}
    }
  ]
}
```

### Resolution

```http
GET /v1/sync/resolutions?cursor=...
```

用于客户端展示最近自动解决审计。

## 自动解决规则

云端应成为正式同步下的最终合并裁判。

建议沿用当前已验证规则：

- 同一 `account_id + source_device_id + client_record_id`：幂等更新。
- 同一 `watermark_uid + asset_fingerprint + revision`：重复忽略。
- 同一 `watermark_uid + asset_fingerprint`，incoming revision 更高：升级版本。
- 同一 `watermark_uid + asset_fingerprint`，incoming revision 更低：忽略旧版本。
- 同一 `watermark_uid`，不同 `asset_fingerprint`：接受为素材变体。

`asset_fingerprint`：

- 优先 `sha256`。
- 取证记录没有 `sha256` 时使用 `extracted_file_hash_hex`。

客户端本地也保留同样规则，用于离线预合并和 UI 去重，但云端裁判结果优先。

## 客户端改造方案

### 移动端

保留：

- `VaultStore`
- `SyncQueueItem`
- 本地版权库
- 下行自动解决审计
- 搜索筛选
- 同步诊断面板

改造：

- `DesktopPairingProfile` 改为 `SyncProfile`。
- `SyncTransportMode.mock/http` 改为 `localOnly/cloud/lanDebug`。
- `DesktopHttpSyncTransport` 改名为 `LanDebugSyncTransport`。
- 新增 `CloudSyncTransport`。
- 设置页主入口改成“登录并开启云同步”。
- 局域网调试入口移动到“高级 / 开发模式”。

移除主路径：

- 桌面地址。
- 配对码。
- 桌面 HTTP 作为默认同步选项。

### 桌面端

保留：

- 本地 SQLite 版权库。
- 本地处理能力。
- 同步事件表。
- 自动解决审计 UI。

改造：

- 增加账户登录。
- 增加云同步队列。
- 增加云端 changes 拉取。
- 设置页主同步入口改为云同步账户状态。

降级：

- `src-tauri/src/sync/server.rs` 作为 `lan_debug_sync`。
- 默认不启动或仅开发模式启动。
- 设置页不把配对码作为正式同步入口。

## UI 调整

### 设置页正式信息架构

```text
账户
  登录状态
  当前账户
  订阅/权益

同步
  本机模式 / 云同步
  上次同步时间
  待上传
  待拉取
  最近错误
  重试

版权库
  本地记录数
  云端记录数
  自动解决审计

高级
  局域网调试同步
  导入/导出本地数据
  清除本机数据
```

### 移动端文案调整

当前“桌面端同步”应改为：

- 主路径：`云同步`
- 开关说明：`登录后可在手机和桌面端同步版权库与取证记录。`
- 高级入口：`局域网调试同步`

当前“配对码”不再出现在默认设置页。

## 迁移计划

### Phase 0：冻结决策

- 文档确认：桌面直连不是正式同步方案。
- UI 标注：当前 HTTP 同步改名为局域网调试同步。
- 不继续在直连方案上做产品化投入。

### Phase 1：客户端抽象重命名

- `DesktopPairingProfile -> SyncProfile`
- `SyncTransportMode.http -> lanDebug`
- 设置页把直连同步移到高级区。
- 版权库、本地队列、审计保持不变。

### Phase 2：账户模型

- 增加登录态模型。
- 增加设备注册模型。
- 增加账户设置页。
- 本地记录增加 `account_id` 可空字段。

### Phase 3：云同步 API MVP

- 实现账户登录。
- 实现设备注册。
- 实现 push batch。
- 实现 pull changes。
- 云端实现自动解决审计。

### Phase 4：双端接入

- 移动端接 `CloudSyncTransport`。
- 桌面端接 `CloudSyncTransport`。
- 局域网调试同步隐藏到高级设置。

### Phase 5：迁移与回滚

- 本地旧记录首次登录后提示“加入云同步”。
- 用户可选择：
  - 仅本机保留。
  - 上传元数据到云端。
  - 导出本地备份。
- 云同步失败不影响本地使用。

|---|---|
| 移动端本地水印 | 保留 |
| 移动端本地版权库 | 保留 |
| 移动端同步队列 | 保留 |
| 移动端自动解决审计 | 保留 |

## 风险

- 云同步需要后端、账号、鉴权和运维，工作量明显大于直连。
- 隐私政策和用户协议必须更新。
- 订阅权益和云同步容量要提前定义。
- 如果云服务不稳定，会影响跨设备体验，所以必须保证本地模式完全可用。

## 推荐下一步

先做 Phase 1：客户端抽象重命名和 UI 降级，把当前“桌面端同步”从主产品路径移到高级/开发模式。这样可以立刻停止错误方向继续固化，同时保留已经做好的联调价值。
