# HiddenShield 商业化与匿名反馈方案

生效目标：后续收费版本

## 0. 基础口径

- 当前版本继续保持免费使用，收费仅作为后续版本的商业化方向。
- 主收费模型是订阅，点数包仅作为后续可选补充，不作为当前主卖点。
- 匿名统计采用“明确告知、默认可见、可随时关闭”的方式，不做隐式采集。
- 用户反馈以低打扰方式进行，成功事件只记轻量统计，失败事件可附带脱敏诊断信息。

## 1. 目标

- 先保持免费期，积累真实使用量、稳定性数据和用户口碑。
- 逐步切换到订阅为主的收费模式，后续可扩展点数包。
- 建立匿名反馈闭环，用于发现 bug、性能瓶颈和核心功能流失点。

## 2. 收费策略

### 2.1 阶段划分

- 阶段 1：免费期
  - 核心功能继续免费开放。
  - 收费入口可先隐藏或弱展示。
  - 主要目标是验证留存、转化和功能使用频次。
- 阶段 2：内部试收费
  - 开放订阅状态管理，但不急于接入真实支付。
  - 验证权益切换、试用、宽限期、过期回收逻辑。
- 阶段 3：正式收费
  - 订阅作为主收费形态。
  - 点数包作为补充方案，后续再开放。

### 2.2 商业模式

- 主形态：订阅。
  - 包月/包年，订阅期内无限次使用。
  - 适合高频创作者和重度用户。
- 预留形态：点数。
  - 每成功处理一张图片或一个视频消耗 1 点。
  - 只做数据结构和接口预留，首发不主推。
- 计量原则：
  - 只有成功完成处理才计数。
  - 失败、取消、崩溃不扣次数。

## 3. 匿名反馈策略

### 3.1 原则

- 匿名统计必须明确告知，并提供清晰的开关控制。
- 可见、可关、可导出说明。
- 不采集文件内容，不采集文件路径，不采集完整 hash，不采集原始媒体 bytes。
- 只采集最小必要诊断信息，用于定位稳定性与体验问题。

### 3.2 采集字段

建议保留以下字段：

- 匿名 install id
- 匿名 session id
- 应用版本
- 功能名
- 成功/失败
- 错误码
- 耗时
- 文件类型桶
- 文件大小桶
- 崩溃堆栈摘要

### 3.3 反馈方式

- 成功事件：轻量上报。
- 失败事件：附带脱敏诊断包。
- 事件先入本地队列，空闲批量上传。
- 断网自动重试。
- 退出前 flush 本地队列。

## 4. 本地数据模型

### 4.1 权益状态

状态至少包括：

- `free`
- `trial`
- `active`
- `grace`
- `expired`

### 4.2 用量账本

- 每条记录代表一次成功处理。
- 记录字段建议包括：
  - 事件 id
  - 发生时间
  - 功能类型
  - 媒体类型
  - 是否成功
  - 计量单位
  - 订阅/套餐归属

### 4.3 匿名事件队列

- 本地持久化事件队列表。
- 支持待发送、已发送、失败重试、过期清理。
- 允许脱敏诊断包与轻量事件共存。

## 5. UI 与合规

- 首启弹窗或 banner 明确说明匿名统计、联网取证与关闭方式。
- 设置页提供：
  - 匿名统计开关
  - 联网开关
  - 订阅入口
  - 崩溃日志导出
  - 使用统计概览
- 法务文案需要同步更新：
  - 隐私政策
  - 用户协议
  - 产品内提示文案

## 6. 风险与边界

- 不做隐式上传原始素材。
- 不做无法关闭的用户行为画像。
- 不在免费期做强制付费拦截。
- 点数包后续可加，但不影响订阅主线。
