# HiddenShield 移动端真机联调手册

本文用于把 Flutter 移动端 App 与桌面端 HiddenShield 在同一局域网内跑通。当前联调范围只覆盖图片/WAV 版权库元数据、取证记录和同步审计，不传输原始媒体文件，本地不做视频盲水印。

## 联调目标

- Android 真机能安装并启动 Flutter App。
- 移动端能保存桌面端地址和配对码。
- 移动端能把写入/取证记录推送到桌面端。
- 移动端能从桌面端拉取版权库和取证增量。
- 移动端设置页“同步诊断”能辅助定位配对、网络、队列和自动解决问题。

## 环境前置

桌面端：

- Windows 开发机已能启动 HiddenShield 桌面端。
- 桌面端同步服务监听 `0.0.0.0:47219`。
- 桌面端设置页可看到移动端同步区域、监听地址、端口和配对码。
- Windows 防火墙允许局域网设备访问 `47219` 端口。

移动端：

- Flutter SDK 可用。
- Android SDK、platform-tools、Gradle 环境可用。
- Rust Android targets 已安装：

```powershell
rustup target list --installed
```

应至少包含：

```text
aarch64-linux-android
armv7-linux-androideabi
x86_64-linux-android
i686-linux-android
```

## 桌面端准备

1. 启动桌面端：

```powershell
npm run tauri dev
```

2. 打开桌面端设置页，确认移动端同步区域显示：

- 监听地址：`0.0.0.0`
- 端口：`47219`
- 当前配对码
- 已收事件数
- 自动解决摘要

3. 获取开发机局域网 IP：

```powershell
ipconfig
```

使用与手机同一 Wi-Fi 网段的 IPv4 地址，例如：

```text
192.168.1.8
```

移动端要填写的桌面地址是：

```text
http://192.168.1.8:47219
```

不要填写 `http://127.0.0.1:47219`，真机上的 `127.0.0.1` 指手机本机，不是电脑。

## 局域网连通性检查

也可以先运行联调诊断脚本：

```powershell
$env:HIDDENSHIELD_SYNC_URL="http://192.168.1.8:47219"
$env:HIDDENSHIELD_PAIRING_CODE="桌面端配对码"
npm run mobile:doctor
```

该脚本会检查 Flutter、移动设备、Rust Android targets、桌面同步健康检查和配对码环境变量。

从电脑本机先确认健康检查可访问：

```powershell
Invoke-WebRequest http://127.0.0.1:47219/api/mobile-sync/v1/health
```

再用电脑的局域网 IP 检查：

```powershell
Invoke-WebRequest http://192.168.1.8:47219/api/mobile-sync/v1/health
```

如果第二条失败，优先检查：

- 手机和电脑是否在同一网络。
- Windows 防火墙是否允许 `47219` 入站。
- 桌面端是否仍在运行。
- 路由器是否开启了 AP 隔离或访客网络隔离。

## Android HTTP 注意项

当前同步协议使用局域网 HTTP。Android 9+ 默认限制明文 HTTP。

开发联调建议二选一：

- Debug 包临时允许 cleartext HTTP。
- 后续把桌面同步服务升级为 HTTPS 或本地安全通道。

当前项目已在 Android `debug` 和 `profile` 构建类型中配置 `usesCleartextTraffic=true` 与 `debug_network_security_config`，用于真机联调局域网 HTTP。`release` 构建不默认放宽明文 HTTP，正式发布前应改为 HTTPS 或更安全的本地配对通道。

## 移动端配对流程

1. 打开移动端 App。
2. 进入“设置”。
3. 打开“桌面端同步”。
4. 填入桌面端地址，例如：

```text
http://192.168.1.8:47219
```

5. 填入桌面端设置页展示的配对码。
6. 点击“保存配对”。
7. 切换同步通道为“桌面 HTTP”。
8. 查看“同步诊断”：

- 桌面地址应为刚填写的局域网地址。
- 配对状态应为“已配对”或后续测试后保持正常。
- 同步通道应为“桌面 HTTP”。
- 最近错误应为“无”。

9. 查看“联调检查”：

- 桌面地址必须是 `http://<电脑局域网 IP>:47219`，不能是 `127.0.0.1` 或 `localhost`。
- 配对码应显示已保存。
- 同步通道应显示“桌面 HTTP”。
- 最近错误应为“无”。

## 推送验证

1. 在移动端“工作台”执行一次图片或 WAV 写入。
2. 进入“版权库”，确认新增记录出现，状态为“待同步”。
3. 进入“设置”，确认待同步队列大于 0。
4. 点击“桌面 HTTP 同步”。
5. 预期结果：

- 移动端待同步队列归零。
- 同步诊断里失败队列为 0。
- 桌面端设置页已收事件数增加。
- 桌面端版权库或同步事件表中出现对应记录。

如失败，查看移动端“同步诊断”的“最近错误”，常见错误含义：

- `desktop address is invalid`：桌面地址格式错误，必须包含 `http://`。
- `pairing code is empty`：未填写配对码。
- `HTTP 401`：请求缺少配对码。
- `HTTP 403`：配对码不匹配或桌面端已生成新配对码。
- `SocketException` / timeout：局域网或防火墙问题。
- `CLEARTEXT communication not permitted`：Android 明文 HTTP 未放行。

失败项修复后，点击“重试失败”即可把失败队列重新发送。

## 拉取验证

1. 确保桌面端已有版权库记录或移动端已成功推送过记录。
2. 移动端进入“设置”。
3. 点击“拉取桌面变更”。
4. 预期结果：

- 移动端“版权库”出现桌面端记录。
- 同步诊断的“上次拉取游标”更新。
- 如出现重复 UID/哈希/revision，移动端不会重复展示，而是写入自动解决审计。
- 设置页“自动解决审计”累计数可能增加，并展示最近一次解决类型。

## 取证记录验证

1. 移动端进入“取证”。
2. 选择图片或 WAV，执行提取。
3. 版权库中应出现来源为“取证”的记录。
4. 同步到桌面端后，桌面端应写入 `sync_evidence_records`。
5. 从桌面端拉取回移动端时，详情页应保留：

- `extracted_timestamp`
- `extracted_device_id_hex`
- `extracted_file_hash_hex`
- `parent_watermark_uid`
- `rewrite_reason`

## 自动解决验证

建议验证以下四类情况：

- 同一桌面 `id` 再次拉取：移动端刷新本地记录。
- 不同桌面 `id`，但同一 `watermark_uid + hash + revision`：移动端忽略重复。
- 同一 `watermark_uid + hash`，更高 revision：移动端升级既有记录。
- 同一 `watermark_uid`，不同 hash：移动端作为素材变体保留。

验证入口：

- 移动端设置页：“自动解决审计”和“同步诊断”。
- 移动端版权库：搜索 UID，查看是否重复展示。
- 移动端版权库详情页：查看 revision、父 UID、重写原因和取证字段。

## 桌面端脚本辅助验证

可用脚本向桌面端同步服务发送测试事件：

```powershell
$env:HIDDENSHIELD_SYNC_URL="http://127.0.0.1:47219"
$env:HIDDENSHIELD_PAIRING_CODE="桌面端配对码"
npm run sync:verify
```

发送取证事件：

```powershell
$env:HIDDENSHIELD_SYNC_EVENT="evidence"
npm run sync:verify
```

发送版权记录和取证记录组成的批量事件：

```powershell
$env:HIDDENSHIELD_SYNC_EVENT="batch"
npm run sync:verify
```

## 常见问题

### 手机无法连接桌面端

优先检查手机和电脑是否同一 Wi-Fi、桌面地址是否填电脑局域网 IP、防火墙是否放行 `47219`。

### 配对失败

确认移动端填写的是桌面端设置页当前配对码。桌面端生成新配对码后，旧配对码会失效，需要在移动端重新保存。

### 推送成功但移动端仍显示待同步

确认当前同步通道是否为“桌面 HTTP”。如果仍失败，查看同步诊断的最近错误和失败队列数。

### 拉取后出现看似重复的记录

先在版权库搜索同一 UID，再打开详情页比较 `revision` 和哈希。当前设计会保留同 UID 不同哈希的素材变体，这不是冲突。

### Debug 能同步，Release 不行

检查 release 包是否允许访问网络、是否允许局域网 HTTP。长期方案应把桌面端同步服务升级到 HTTPS 或更安全的本地配对通道。

## 后续建议

下一步应补 Android debug cleartext 网络配置和联网权限策略说明，确保真机能直接访问局域网 HTTP；Release 版再单独设计 HTTPS/安全通道方案。
