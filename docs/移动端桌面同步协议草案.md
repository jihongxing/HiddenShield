# HiddenShield 移动端桌面同步协议草案

本文定义移动端与桌面端同步服务的第一版 HTTP 协议骨架。当前阶段移动端已有 `DesktopHttpSyncTransport`，桌面端已有最小 HTTP stub，并会校验桌面端当前配对码，可用于端到端联调。

## 目标

- 移动端只同步版权库元数据、取证记录和重写链路。
- 默认不传输原始图片、音频或视频文件。
- 同步队列可重试，失败项保留 `attempts` 和 `last_error`。
- 协议保持简单，后续可以替换为 WebSocket 或局域网发现。

## 配对信息

移动端保存：

- `desktop_address`：桌面端同步服务地址，例如 `http://192.168.1.8:47219`。
- `pairing_code`：桌面端生成的一次性或短期有效配对码。
- `status`：`unpaired`、`paired`、`connecting`、`failed`。

首版请求通过 HTTP header 传递配对码。桌面端会将该值与设置页展示的当前配对码比对，不匹配时拒绝写入：

```http
X-HiddenShield-Pairing-Code: 123456
```

## 批量推送同步事件

移动端优先使用批量接口。桌面端保留单条接口用于兼容旧客户端。

```http
POST /api/mobile-sync/v1/queue-batch
Content-Type: application/json
X-HiddenShield-Pairing-Code: 123456
```

请求体：

```json
{
  "items": [
    {
      "queueId": "1700000000000-upsertVaultRecord",
      "recordId": "1700000000000",
      "operation": "upsertVaultRecord",
      "payloadType": "vault_record",
      "payload": {}
    }
  ]
}
```

成功响应：

```json
{
  "ok": true,
  "accepted": 1,
  "acceptedQueueIds": ["1700000000000-upsertVaultRecord"]
}
```

当前批量接口限制最多 100 条。空批次或超过限制会返回 `400`，任意一条写入失败会返回 `500`，移动端会将本批次队列项标记为失败并等待下次重试。

## 移动端错误处理

- `2xx`：视为同步成功。
- 非 `2xx`：队列项标记为 `failed`，`last_error` 记录状态码和响应摘要。
- 网络异常、超时、JSON 无法编码：队列项标记为 `failed`。
- 当前移动端不会删除已成功队列项，只更新为 `synced`，便于后续审计。
- 移动端设置页可选择“本地模拟”或“桌面 HTTP”。未保存桌面地址和配对码时不能启用 HTTP 模式。

## 拉取桌面端变更

移动端可按时间游标拉取桌面端版权库和取证记录增量。当前只返回元数据，不返回原始媒体文件。

```http
GET /api/mobile-sync/v1/changes?since=2026-06-16T12:00:00.000Z
X-HiddenShield-Pairing-Code: 123456
```

成功响应：

```json
{
  "ok": true,
  "nextSince": "2026-06-16T12:10:00.000Z",
  "changes": [
    {
      "id": "desktop-42",
      "desktop_id": 42,
      "kind": "image",
      "title": "work.png",
      "watermark_uid": "preview-img-abc",
      "revision": 1,
      "sha256": "abc123",
      "parent_watermark_uid": null,
      "rewrite_reason": null,
      "source": "desktop",
      "sync_status": "synced",
      "created_at": "2026-06-16T12:10:00.000Z"
    },
    {
      "id": "desktop-evidence-mobile-record-1",
      "kind": "image",
      "title": "suspect.png",
      "watermark_uid": "preview-img-abc",
      "revision": 2,
      "sha256": null,
      "parent_watermark_uid": "preview-img-parent",
      "rewrite_reason": "authorized rewrite",
      "extracted_timestamp": 1780000000,
      "extracted_device_id_hex": "090a0b0c",
      "extracted_file_hash_hex": "abcd",
      "source": "verify",
      "sync_status": "synced",
      "created_at": "2026-06-16T12:11:00.000Z"
    }
  ]
}
```

移动端合并策略：

- 本地 ID 使用 `desktop:<id>`，避免覆盖手机本地记录。
- 拉取记录标记为 `synced`。
- `source = verify` 的记录保存为移动端取证线索，并保留 `extracted_*` 字段。
- 同一桌面 `id` 再次拉取时，移动端刷新本地记录，并写入 `record_replaced` 审计。
- 不同桌面 `id` 但同一 `watermark_uid + asset_fingerprint + revision` 已存在时，移动端忽略重复展示，并写入 `duplicate_ignored` 审计。
- 同一 `watermark_uid + asset_fingerprint` 但拉取 revision 更高时，移动端升级既有记录并写入 `revision_upgraded` 审计；拉取 revision 更低时忽略并写入 `stale_revision_ignored` 审计。
- 同一 `watermark_uid` 但 `asset_fingerprint` 不同时，移动端保留为同一水印 UID 的素材变体，并写入 `variant_accepted` 审计。
- `asset_fingerprint` 优先使用 `sha256`；取证记录没有 `sha256` 时使用 `extracted_file_hash_hex`。
- `nextSince` 持久化到移动端 `sync_profile.last_desktop_pull_since`，作为下一次拉取游标。

移动端本地审计表：

```sql
mobile_sync_resolutions(
  id TEXT PRIMARY KEY,
  resolved_at INTEGER NOT NULL,
  resolution_type TEXT NOT NULL,
  reason TEXT NOT NULL,
  incoming_record_id TEXT NOT NULL,
  existing_record_id TEXT,
  watermark_uid TEXT NOT NULL,
  existing_hash TEXT,
  incoming_hash TEXT,
  existing_revision INTEGER,
  incoming_revision INTEGER NOT NULL,
  inserted_record_id TEXT
)
```

## 桌面端当前实现

桌面端 Tauri 后端启动时会拉起轻量 HTTP stub：

- `GET /api/mobile-sync/v1/health` 返回桌面设备标识和协议版本。
- `GET /api/mobile-sync/v1/changes?since=...` 校验配对码并返回桌面端版权库和取证记录增量。
- `POST /api/mobile-sync/v1/queue-item` 和 `POST /api/mobile-sync/v1/queue-batch` 校验配对码 header 是否存在且与桌面端当前配对码一致，解析 JSON 并写入 SQLite。
- `operation = upsertVaultRecord` 且 `payloadType = vault_record` 时，桌面端会把移动端版权记录合并进 `vault_records`。
- 同一 `watermark_uid + original_hash + revision` 已存在时只保留同步事件，不重复插入版权记录。
- 同一 `watermark_uid` 但 `original_hash` 不同时，桌面端自动接收为同一水印 UID 的素材变体，并写入 `sync_resolutions` 审计记录。
- 同一 `watermark_uid + original_hash` 但 `revision` 不同时，桌面端自动保留更高 revision；低版本同步会被视为过期事件并写入 `sync_resolutions` 审计记录。
- `operation = upsertEvidenceRecord` 时，桌面端会把移动端取证命中写入 `sync_evidence_records`；同一 `mobile_record_id` 重传时更新该取证记录，同时保留每次同步事件。
- 桌面端设置页展示监听地址、端口、配对码、已收事件数、最近事件时间和自动解决摘要。
- 桌面端设置页可生成新的移动端配对码。

当前写入表：

```sql
sync_events(
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  received_at TEXT NOT NULL,
  queue_id TEXT NOT NULL,
  record_id TEXT NOT NULL,
  operation TEXT NOT NULL,
  payload_type TEXT NOT NULL,
  payload_json TEXT NOT NULL
)
```

```sql
sync_evidence_records(
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  mobile_record_id TEXT NOT NULL UNIQUE,
  received_at TEXT NOT NULL,
  suspect_file_name TEXT NOT NULL,
  asset_kind TEXT NOT NULL,
  watermark_uid TEXT NOT NULL,
  revision INTEGER NOT NULL,
  parent_watermark_uid TEXT,
  rewrite_reason TEXT,
  extracted_timestamp INTEGER,
  extracted_device_id_hex TEXT,
  extracted_file_hash_hex TEXT,
  payload_json TEXT NOT NULL
)
```

```sql
sync_resolutions(
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  resolved_at TEXT NOT NULL,
  queue_id TEXT NOT NULL,
  mobile_record_id TEXT NOT NULL,
  resolution_type TEXT NOT NULL,
  reason TEXT NOT NULL,
  watermark_uid TEXT NOT NULL,
  desktop_record_id INTEGER,
  desktop_hash TEXT,
  mobile_hash TEXT,
  desktop_revision INTEGER,
  mobile_revision INTEGER,
  inserted_record_id INTEGER,
  payload_json TEXT NOT NULL
)
```

当前桌面端已支持移动端版权记录、取证记录、自动解决审计入库，以及桌面版权库和取证记录到移动端的增量拉取。移动端已支持下行增量的确定性去重、版本升级、素材变体接收和本地审计。媒体文件同步和多设备游标管理仍待实现。

## 端到端联调

启动桌面端后，可发送一条测试事件：

```bash
HIDDENSHIELD_PAIRING_CODE=123456 npm run sync:verify
```

如需指定桌面端地址：

```bash
HIDDENSHIELD_SYNC_URL=http://127.0.0.1:47219 HIDDENSHIELD_PAIRING_CODE=123456 npm run sync:verify
```

如需同时检查 SQLite 落库，传入桌面端 `vault.db` 路径：

```bash
HIDDENSHIELD_PAIRING_CODE=123456 HIDDENSHIELD_VAULT_DB=/path/to/vault.db npm run sync:verify
```

Windows PowerShell 示例：

```powershell
$env:HIDDENSHIELD_PAIRING_CODE="123456"
$env:HIDDENSHIELD_VAULT_DB="C:\path\to\vault.db"
npm run sync:verify
```

发送取证事件：

```powershell
$env:HIDDENSHIELD_SYNC_EVENT="evidence"
npm run sync:verify
```

发送版权记录和取证记录组成的批量事件：

```powershell
$env:HIDDENSHIELD_SYNC_EVENT="batch"
npm run sync:verify
```
