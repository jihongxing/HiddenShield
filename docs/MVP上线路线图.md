# 隐盾 MVP 上线路线图

代码 MVP 已完成，以下为从"能跑"到"能卖"的四阶段执行计划。

---

## 阶段一：打包封版与 Dogfooding 测试

**目标：** 在纯净"野生环境"下验证核心链路可靠性。

### 环境准备
- [ ] Windows 纯净机（无开发环境、无 FFmpeg）安装测试
- [ ] macOS 纯净机安装测试
- [ ] 验证 FFmpeg / ffprobe 预装指引、PATH 探测与环境自检在两平台均正常

### 极端输入测试
- [ ] 10GB+ 超大 4K 视频：进度条平滑、内存稳定
- [ ] 生僻字/Emoji 文件名（如 `【第①集】🎬测试.mp4`）
- [ ] 压制 50% 时断网 → 验证容错与临时文件清理
- [ ] 压制 50% 时合上笔记本 → 验证休眠抑制与恢复
- [ ] 磁盘空间不足时的友好提示

### 输出质量盲测
- [ ] 隐盾压制 vs 剪映直出 vs 原始上传：画质 A/B 盲测
- [ ] 音频水印"绝对无感"验证（耳机 + 音箱双重测试）
- [ ] 图片 LSB 水印不影响视觉质量验证

### 打包配置
- [ ] Tauri 打包 Windows `.msi` / `.exe` 安装包
- [ ] Tauri 打包 macOS `.dmg` 安装包
- [ ] 应用图标、名称、版本号确认
- [ ] Windows EV 代码签名（防杀软误报）
- [ ] macOS 公证 (Notarization)

---

## 阶段二：种子用户内测 (Alpha Release)

**目标：** 5-10 个真实创作者验证产品价值，收集口碑素材。

### 种子用户招募
- [ ] B站影视解说 UP 主 3-5 人
- [ ] 小红书/抖音短剧剪辑手 3-5 人
- [ ] 筛选标准：对画质有极高要求、有被盗稿经历

### 旁观测试 (Usability Testing)
- [ ] 发安装包，不附操作说明
- [ ] 观察能否 1 分钟内完成"拖入 → 选平台 → 导出"
- [ ] 记录所有卡点和疑惑
- [ ] 收集改进建议

### 核心资产收集
- [ ] 用户实测对比截图（隐盾 vs 直传）
- [ ] 用户真实评价/推荐语
- [ ] 平台画质提升的量化数据（码率、清晰度对比）

---

## 阶段三：宣发物料准备 (Marketing)

**核心策略：** 痛点营销 > 技术营销。卖"画质提升"，不卖"盲水印"。

### 对比物料制作
- [ ] 核心对比视频：iPhone HDR 直传抖音（发灰变糊） vs 隐盾色彩映射后上传（高清通透）
- [ ] 对比图组：同一视频在三平台的画质差异
- [ ] 30 秒产品演示 GIF/短视频

### 官网/落地页
- [ ] 核心标语定稿：「别让平台压缩毁了你的画质。一键输出全网最优规格，附带军工级隐形防盗水印。」
- [ ] 极简单页设计：痛点 → 方案 → 对比 → 下载
- [ ] 免登录直接下载链接
- [ ] 种子用户真实评价展示

### 分发渠道
- [ ] B站投稿：面向影视区 UP 主的画质提升教程
- [ ] 小红书笔记：面向短视频创作者的防盗水印科普
- [ ] 微信公众号/知乎专栏：技术深度文章引流

---

## 阶段四：商业化与法务基建

### 法务合规
- [x] EULA 用户协议实装（首次启用联网取证 / 遥测前展示并确认）
- [ ] 免责声明内容：不对维权结果的连带法律责任负责
- [x] 隐私政策（本地优先，联网取证 / 遥测单独告知）
- [ ] 软件著作权登记

### 支付通道预埋
- [ ] V1.0 免费引流版 + "升级 Pro 版"按钮（测试付费意愿）
- [ ] 或"打赏/请开发者喝咖啡"入口
- [ ] 记录点击率作为付费意愿数据

### Pro 版差异化（V2.0 规划）
- [ ] 批量压制（免费版限单文件）
- [ ] 更多平台预设（YouTube、Twitter/X、微信视频号）
- [ ] 自定义压制参数
- [ ] 优先技术支持

---

## 里程碑时间线（建议）

| 阶段 | 预计周期 | 关键产出 |
|------|----------|----------|
| 阶段一 | 1-2 周 | 稳定安装包 + 测试报告 |
| 阶段二 | 2-3 周 | 种子用户反馈 + 口碑素材 |
| 阶段三 | 1-2 周 | 官网上线 + 宣发物料就绪 |
| 阶段四 | 并行推进 | 法务资料固化 + 支付预埋 |

**首次公开发布 (Public Beta)：** 阶段一~三完成后即可启动。
