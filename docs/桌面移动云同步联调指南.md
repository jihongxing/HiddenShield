# HiddenShield 桌面 / 移动 / 云同步联调指南

本文用于同时预览桌面端、Flutter 移动端和最小真实云同步后端。正式同步路径是：

```text
桌面端 Tauri -> HiddenShield Cloud Sync API <- Flutter 移动端
```

局域网直连桌面端只保留为高级调试 / 开发联调能力，不作为正式产品同步方案。

## 一键启动

默认启动三部分：

- 云同步后端：`http://127.0.0.1:43188`
- 桌面端：Tauri 开发模式
- 移动端：Flutter Web 预览，默认 `http://127.0.0.1:43189`

```powershell
npm run dev:stack
```

桌面端必须通过 Tauri 预览，不使用 `vite preview` 代替。Flutter Web 只用于快速看 UI 和同步流程，真机能力仍以 Android / iOS 运行结果为准。

## 常用启动方式

只启动云后端和桌面端：

```powershell
npm run dev:stack -- --no-mobile
```

启动云后端、桌面端和指定 Flutter 真机：

```powershell
cd mobile_app
flutter devices
cd ..
npm run dev:stack -- --mobile-device=<flutter-device-id>
```

只启动云后端，便于单独跑契约测试：

```powershell
npm run dev:stack -- --no-desktop --no-mobile
```

验证云同步契约：

```powershell
npm run cloud:contract
```

验证桌面 / 移动双设备云同步回环：

```powershell
npm run cloud:e2e
```

## 系统配置

桌面端读取：

```text
config/hiddenshield.system.json
```

移动端读取打包资产：

```text
mobile_app/assets/hiddenshield.system.json
```

两者当前都指向：

```text
http://127.0.0.1:43188
```

用户设置页不提供手动填写云服务地址。后续如果切换测试环境，应通过系统配置文件调整，而不是让用户自己设置。

## 联调验收路径

1. 启动：

```powershell
npm run dev:stack
```

2. 确认云后端健康：

```powershell
Invoke-WebRequest http://127.0.0.1:43188/v1/health
```

3. 桌面端设置页进入“账户与云同步”：

- 输入邮箱或手机号。
- 继续使用账户。
- 确认账户、工作区、创作者档案和权益快照出现。

4. Flutter 移动端设置页进入“账户与云同步”：

- 输入同一邮箱或手机号。
- 继续使用账户。
- 确认移动端显示同一账户、工作区、创作者档案和权益快照。

5. 在桌面端写入一条图片或音频版权记录：

- 记录应进入本地版权库。
- 云同步队列应出现待上传项。
- 执行刷新 / 同步后应上传到云端。

6. 在移动端拉取云端变更：

- 版权库应出现桌面端同步过来的记录。
- 不应出现原始媒体文件、产物媒体文件或本地文件路径。

7. 反向验证：

- 移动端写入或取证生成记录。
- 执行云同步。
- 桌面端拉取云端变更。
- 桌面端版权库能看到对应元数据。

## 数据边界

默认同步：

- 账户摘要。
- 默认工作区。
- 设备登记摘要。
- 默认创作者档案。
- 订阅权益快照。
- 版权库元数据。
- 取证记录元数据。
- 水印 UID、revision、父级 UID、重写原因。
- 文件 hash / 取证 hash。
- 自动解决审计。

默认不同步：

- 原始图片。
- 加水印后的图片。
- 原始音频。
- 加水印后的音频。
- 原始视频。
- 加水印后的视频。
- 本地文件路径。

## 常见问题

### Flutter Web 白屏

先确认控制台是否显示资源或 Dart 编译错误。当前 Flutter Web 使用 preview bridge，不加载 Rust FFI；如果 Web 白屏，优先跑：

```powershell
cd mobile_app
flutter analyze
flutter test
flutter run -d chrome --web-hostname 127.0.0.1 --web-port 43189
```

### 桌面端打开的是网页而不是应用

桌面端预览必须使用：

```powershell
npm run tauri:dev
```

或：

```powershell
npm run dev:stack -- --no-mobile
```

不要使用 `npm run preview` 判断桌面能力，因为 Tauri IPC、Rust 后端、文件系统和本地数据库都需要 Tauri 容器。

### 云同步失败

优先检查：

- `npm run cloud:contract` 是否通过。
- `config/hiddenshield.system.json` 和 `mobile_app/assets/hiddenshield.system.json` 是否指向同一云后端。
- 桌面端和移动端是否都已经“继续使用账户”。
- 本地队列是否有失败项和最近错误。

## 推荐下一步

下一步应把这份联调流程固化为 CI 可跑的最小端到端契约：启动云后端后，用脚本模拟桌面端和移动端分别登录同一账户、各上传一条记录、互相拉取并断言不会同步媒体文件和本地路径。
