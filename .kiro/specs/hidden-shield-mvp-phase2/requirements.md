# 需求规格：隐盾 MVP Phase 2 — 体验强化与商业闭环

## 概述

MVP Phase 1 完成了核心功能闭环（压制 + 水印 + 金库 + 取证）。Phase 2 聚焦于"第一次用就觉得值"和"第二次还会主动打开"，实现 8 个功能方向。

---

## 功能 1：首次启动自检 + 引导

### FR-1.1 FFmpeg 环境自检
- 应用启动时自动检测 FFmpeg 可用性、硬件编码器状态
- 检测磁盘可用空间（输出目录所在分区）
- 检测输出目录写入权限
- 将自检结果汇总为结构化状态对象返回前端

### FR-1.2 自检结果展示
- 在工作台顶部展示环境状态卡片（FFmpeg ✓/✗、GPU 编码 ✓/✗、磁盘空间、输出目录）
- 异常项高亮显示并提供修复建议（如"点击下载 FFmpeg"、"磁盘空间不足，建议清理"）
- 所有项通过时折叠为一行摘要

### FR-1.3 素材异常前置提醒
- 导入文件后，根据 SourceMeta 自动检测并提示：
  - "当前素材是 HDR，将自动进行色调映射"
  - "当前是横屏素材，发抖音/小红书将转为竖屏（加黑边或裁剪）"
  - "当前素材无音轨，将跳过音频水印保护"
  - "当前素材时长超过 30 分钟，预计处理时间较长"
- 提醒以非阻塞方式展示在源文件画像区域

## 功能 2：处理完成页 + 前后结果对比

### FR-2.1 处理完成结果页
- 压制完成后展示结果摘要页面（替代当前简单的"任务完成"提示）
- 清晰列出每个平台的输出文件路径、文件大小、分辨率
- 显示处理耗时、使用的编码器
- 明确提示"版权保护已启用 ✓"及水印 UID

### FR-2.2 处理前后对比
- 展示源文件 vs 输出文件的关键指标对比：
  - 分辨率变化（如 3840x2160 → 1080x1920）
  - 文件大小变化（如 824MB → 45MB，压缩率 94.5%）
  - 帧率变化
  - 色彩空间变化（HDR → SDR）
- 以表格或卡片形式展示

### FR-2.3 一键操作
- "打开输出目录"按钮：调用系统文件管理器打开输出文件夹
- "复制文件路径"按钮：将输出文件路径复制到剪贴板

## 功能 3：平台推荐智能化

### FR-3.1 素材类型自动识别与推荐
- 根据源视频宽高比自动判断：横屏（16:9）/ 竖屏（9:16）/ 方形（1:1）
- 根据时长判断：短视频（< 3min）/ 中视频（3-20min）/ 长视频（> 20min）

### FR-3.2 平台组合自动推荐
- 竖屏短视频 → 默认勾选抖音 + 小红书
- 横屏中长视频 → 默认勾选 B站
- 竖屏中视频 → 默认勾选抖音 + 小红书 + B站
- 用户可手动覆盖推荐

### FR-3.3 策略自动推荐
- 横屏发竖屏平台 → 默认推荐"加黑边"，提示"智能裁剪可能丢失画面边缘"
- HDR 素材 → 提示"将自动进行色调映射以确保平台兼容"
- 有 GPU → 默认推荐"极速 GPU"模式

### FR-3.4 风险提示
- "横屏发抖音/小红书展示面积较小，建议使用智能裁剪"
- "当前素材帧率 60fps，B站将保留 60fps，其他平台降为 30fps"

### FR-4.2 存证卡片交互
- 一键复制存证摘要（纯文本格式）
- 在版权库时间线中高亮显示"本次新增记录"

### FR-4.3 存证卡片展示
- 在处理完成页中嵌入展示
- 在版权库列表中以卡片形式展示每条记录的核心存证信息

## 功能 5：取证模块轻量闭环

### FR-5.1 取证结果分级展示
- 三类结果清晰区分：
  - ✅ 已命中：绿色高亮，展示匹配的原始记录详情
  - ⚠️ 疑似命中：黄色提示，展示置信度和可能原因
  - ❌ 未命中：灰色展示，解释可能原因

### FR-5.2 命中后关联原记录
- 命中后直接展示版权库中的原始记录信息
- 提供"跳转到版权库查看完整记录"的链接/按钮

### FR-5.3 取证摘要
- 提供简版取证摘要文本（不做复杂 PDF）
- 包含：检测时间、疑似文件路径、匹配结果、水印 UID、置信度、免责声明
- 一键复制摘要

### FR-5.4 未命中原因解释
- 当未命中时，提供可能原因说明：
  - "该文件可能经过深度篡改（重编码、裁剪等）"
  - "该文件可能非本机处理的作品"
  - "音频轨道可能已被替换"

## 功能 9：崩溃收集与遥测 (Crash Telemetry)

### FR-9.1 Rust Panic 捕获与上报
- 设置全局 panic hook，捕获 panic 堆栈信息
- 将 panic 信息序列化后通过轻量 HTTP 接口上报（或集成 Sentry Rust SDK）
- 仅上报：panic 消息、堆栈帧、OS 版本、应用版本、匿名设备 ID
- 绝不上报：用户文件名、本地路径、视频内容相关信息

### FR-9.2 FFmpeg 进程崩溃捕获
- FFmpeg 子进程非正常退出时（exit code ≠ 0），记录 exit code 和 stderr 最后 20 行
- 上报内容：exit code、stderr 摘要（脱敏后）、使用的编码器、输入格式（仅 codec/resolution，不含文件名）
- 本地同时写入日志文件供用户自助排查

### FR-9.3 遥测开关与隐私声明
- 提供全局遥测开关（设置页面），默认开启
- 首次启动时以非阻塞 banner 告知用户"隐盾会上报崩溃信息以改善稳定性，不含任何个人文件信息"
- 用户关闭后完全停止上报

### FR-9.4 本地崩溃日志
- 无论遥测开关状态，所有 panic 和 FFmpeg 崩溃均写入本地日志文件
- 日志路径：`{app_data_dir}/logs/crash.log`
- 提供"导出日志"按钮供用户反馈时附带

## 功能 10：自动更新 (Auto-Updater)

### FR-10.1 Tauri Updater 配置
- 在 `tauri.conf.json` 中启用 updater 插件
- 配置更新检查端点 URL（支持 GitHub Releases JSON 格式）
- 配置公钥用于签名验证

### FR-10.2 启动时静默检查更新
- 应用启动后 5 秒静默调用 `check` API
- 有新版本时在界面顶部展示非阻塞更新横幅
- 横幅内容："发现新版本 vX.Y.Z，一键升级"
- 用户点击后下载并安装，安装完成后提示重启

### FR-10.3 更新进度展示
- 下载过程中展示进度条
- 下载失败时提示"更新失败，请稍后重试"并提供手动下载链接
- 不阻塞用户正常使用

### FR-10.4 更新策略
- 普通更新：用户确认后安装
- 强制更新（服务端标记）：弹出模态框，必须更新后才能继续使用（预留能力，MVP 不启用）

## 功能 11：卸载清理 (Uninstall Cleanup)

### FR-11.1 Windows NSIS 卸载脚本
- 自定义 NSIS 卸载脚本，在卸载时清理：
  - `%APPDATA%/com.hiddenshield.desktop/` 目录（FFmpeg、SQLite 数据库、日志）
  - 卸载前弹出确认："是否同时删除版权库数据？（删除后不可恢复）"
  - 用户选择保留时仅清理 FFmpeg 缓存和日志，保留 vault.db

### FR-11.2 应用内清理功能
- 设置页面提供"清除所有数据"按钮
- 点击后二次确认，确认后删除：FFmpeg 缓存、SQLite 数据库、日志文件
- 操作完成后提示"所有数据已清除，可安全卸载"
- 展示当前数据占用空间（FFmpeg 大小 + 数据库大小 + 日志大小）

### FR-11.3 macOS 清理引导
- 设置页面展示数据存储路径提示
- 提供"清除所有数据"按钮（同 FR-11.2）
- 提示文案："macOS 拖拽卸载不会清理应用数据，建议卸载前点击此按钮"

## 功能 12：CI/CD 签名与分发 (Code Signing & Distribution)

### FR-12.1 GitHub Actions 构建流水线
- 创建 `.github/workflows/release.yml`
- 触发条件：推送 `v*` tag
- 构建目标：Windows x86_64 (NSIS installer) + macOS universal (dmg)
- 使用 `tauri-apps/tauri-action` 官方 Action

### FR-12.2 Windows 代码签名
- 从 GitHub Secrets 读取 EV 证书（PFX + 密码）
- 构建完成后对 .exe 和 .msi 进行 Authenticode 签名
- 包含时间戳（防止证书过期后签名失效）

### FR-12.3 macOS 签名与公证
- 从 GitHub Secrets 读取 Apple Developer 证书（P12）和 Notarization 凭据
- 构建完成后进行 codesign + notarize + staple
- 确保 Gatekeeper 不拦截

### FR-12.4 Release 产物发布
- 构建产物自动上传到 GitHub Releases
- 同时生成 `update.json`（供 Tauri Updater 消费）
- Release Notes 从 tag message 或 CHANGELOG 自动提取
